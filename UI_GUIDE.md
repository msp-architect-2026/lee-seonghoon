# 🎨 Common UI/UX Guide: Personal Color AI Analysis

##  개요
본 문서는 'Personal Color AI Analysis' 모바일 웹 애플리케이션의 프론트엔드 화면 개발 및 UI 디자인을 위한 공통 가이드라인입니다. 본 애플리케이션은 **프라이버시 중심(Privacy-First)**의 지능형 분석과 더불어, 사용자에게 지루함 없는 경험(Zero-Boredom)과 시각적 타격감(Impactful Curation)을 제공하는 것을 최우선 목표로 합니다. 
본 가이드는 기획자, 디자이너, 프론트엔드 개발자(Next.js) 간의 일관된 산출물을 위해 작성되었습니다.

##  메뉴 구조도 (IA - Information Architecture)
현재 유저 플로우를 기반으로 도출된 핵심 화면 구조입니다.

* **`/login`** : 소셜 로그인 (Onboarding)
* **`/capture`** : 스마트 캡처 (카메라 권한, 얼굴 가이드라인, 실시간 조명 피드백)
* **`/analyzing`** : 인터랙티브 로딩 (v0.app 기반 안면 스캔 애니메이션)
* **`/result/[id]`** : 진단 결과 (70% 컬러 팔레트 뷰, 메이크업/헤어 레시피 큐레이션)
* **`/mypage/history`** : 과거 진단 이력 조회 목록

##  UI 가이드 세부 항목

### 1. 화면 정의 (Screen Definition)
본 프로젝트는 **Mobile-First 웹 애플리케이션**입니다.
* **Layout Breakpoint:** 모바일 환경(MO)을 최우선 기준으로 설계함.

### 2. 레이아웃 (Layout)

* **Camera Area (캡처 화면):** 사용자 몰입을 위해 상/하단 네비게이션을 최소화하고 화면 전체를 뷰파인더로 사용합니다. 중앙에 얼굴을 맞출 수 있는 오버레이 가이드라인이 존재합니다.
* **Result Area (결과 화면):** 시각적 카타르시스를 위해 **도출된 퍼스널 컬러 팔레트가 화면 전체 영역의 70% 이상을 차지**하도록 상단 영역을 크게 할당합니다. 하단 30% 영역은 스크롤이 가능한 Bottom Sheet 형태로 메이크업/헤어 큐레이션을 배치합니다.

### 3. 인터랙션 (Interaction)
* **전역 상태 연동 (Zustand):** 카메라의 조명 상태, AI 분석 진행률 등의 상태 값에 따라 UI 컴포넌트(버튼 활성/비활성, 경고 표시)가 즉각적으로(Real-time) 반응해야 합니다.
* **v0.app 컴포넌트 활용:** 복잡한 애니메이션은 v0.app을 통해 추출된 Tailwind 기반 컴포넌트를 사용하여 프레임 드랍 없이 부드럽게 전환되도록 구성합니다.

### 4. 컴포넌트 (Component)
* **Button:** * `Primary Button`: 촬영 버튼, 다음 단계 이동 등 핵심 액션 (가장 높은 대비와 크기)
  * `Disabled Button`: 조명 불량 등 조건 미충족 시 액션 방지
* **Color Palette:** 헥스(Hex) 코드를 직접 렌더링하는 동적 색상 블록 컴포넌트.

### 5. 로딩 UX (Zero-Boredom Loading)

* **원칙:** 시스템(FastAPI)에서 무거운 AI 연산이 진행되는 1~2초간 **단순한 스피너 아이콘의 사용을 엄격히 금지**합니다.
* **구현:** 서버로부터 `202 Accepted` 응답을 받으면, 프론트엔드는 즉시 사용자의 캡처된 이미지를 배경으로 두고 그 위를 훑고 지나가는 **'화려한 레이저 스캔 애니메이션'**을 재생합니다.

### 6. 에러 페이지 및 예외 처리 (Error Handling)
* **카메라 권한 거부 시:** 카메라 접근 권한 필요 안내 팝업 및 설정 이동 유도.
* **분석 실패 시:** 조명 문제 또는 얼굴 랜드마크 추출 실패 시, "분석에 실패했습니다. 밝은 곳에서 정면을 보고 다시 시도해주세요." 문구와 함께 캡처 화면으로 복귀합니다.

## 📝 세부 규칙 정의

### 1. 버튼 및 UX Writing 규칙
* **명확하고 직관적인 톤앤매너**를 유지합니다.
* **결과 큐레이션 문구:** 단순히 "이 색이 어울립니다"가 아닌, 전문적이고 시각적인 묘사를 사용합니다.
  * *예시:* "모브 핑크 (Mauve Pink)", "애쉬 그레이지 (Ash Greige) - 애쉬 6 : 베이지 3 : 블루 1 (탈색 1~2회 권장)"

### 2. 알림 (Notification) 및 스마트 피드백 프로세스
퍼스널 컬러 진단의 정확도를 위해 사용자의 환경을 능동적으로 통제합니다.

* **Trigger:** 프론트엔드 단에서 실시간 명도/대비 분석 결과가 기준치 미달일 경우.
* **UI Action:** 1. 촬영(Capture) Primary 버튼이 실시간으로 비활성화(`Disabled`) 됩니다.
  2. 화면 중앙 또는 하단에 Toast 또는 오버레이 형태의 경고 메시지가 즉시 노출됩니다.
* **UX Writing:** *"조명이 너무 어둡습니다. 밝은 곳으로 이동해 주세요."* (문제를 명확히 짚고, 해결 행동을 촉구)
* **Trigger:** 프론트엔드 단에서 실시간 명도/대비 분석 결과가 기준치 미달일 경우.
